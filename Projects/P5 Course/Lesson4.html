Conditions and expressions

In a language you use a condition when you use words as if and but, when, then, after, before, under, above, etc. In code we can use if
If is a powerfull function in javascript. You get control over the execution of your code.

                 if(expression){
                     //do this
                 }
                 else{
                    //do this
                 }

Now you get control over your car so you can improve this game.
  -start when you click on your mouse.
  -stop after you reached the finish line.
Between the parenthesis there is an expression, that is true or false.
This expression can be:
   - a variable that stores the value true or valse.
   - a logical expression like: 

                a == b (a is equal to b)
                a > b ( a is greater then b)
                a < b ( a is smaller then b)
                a >= b ( a is greater then or equal to b)
                 a <= b ( a is smaller then or equal to b)
                (a and b are variable, the sine between a and b are operators.
                the result of an expression is always true or false.)
   - a combination with logical expressions 
                a > b && c == d  (a is greater then b AND c is equal to d. this is true if both expressions are true.)
                a > b || c == d  (a is greater then b OR  c is equal to d. this is true if one of the expressions is true.)
   - a function (a function can return with a value, but I will discus this later in the course.)

So this is a lot to comprehend. Let's have a look at some examples. Also on the Example page you can find several examples. 

              if(mouseIsPressed){
                  // start your engines!
              }
or if you use a keyboard. 
              if(keyIsPressed){
                  // start your engines!
              }

x = 2 means the variable x , a little box with the name x, has the value 2 inside him now. this is an assignment not an expression.
x == 2 means, if the variable x has the value 2, then this is true if not this expression is false. You never can use an assignment after if. This is a mistake every programmer will make, because the difference is so small.
